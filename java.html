Definitions

  • argument: A value that you provide when you call a method. This value must have the type that the method expects.
  • break : The break keyword is used to break out a for loop, a while loop or a switch block.
  • bug: An error in a program.
  • assignment: A statement that gives a value to a variable.
  • concatenation : joining two strings using the + operator
  • declaration: A statement that creates a new variable and specifies its type.
  • debugging: The process of finding and removing errors.
  • expression: A combination of variables, operators, and values that represents a single value.
  • keyword : Java has a set of keywords that are reserved words that cannot be used as variables, methods, classes, or any other identifiers.
  • floating-point : A data type that represents numbers with an integer part and a fractional part. In Java, the default floating-point type is double. The other data type that can be used for floating point numbers is float.
  • initialize: To assign a variable for the first time.
  • identifier: An identifier is nothing but a user defined name assigned to an element in a program. For example : String sName; sName being the identifier.
  • immutable : once created, their values cannot be changed example being a string variable.
  • literal: A value that appears in source code. For example, "Hello" is a string literal, and 74 is an integer literal.
  • method: A named sequence of statements.
  • operator: A symbol that represents a computation like addition, multiplication, or string concatenation.
  • newline: A special character signifying the end of a line of text. Also known as “line ending”, “end of line” (EOL), or “line break”
  • parameter: parameter: A piece of information that a method requires before it can run. Parameters are variables: they contain values and have types.
  • print statement: A statement that causes output to be displayed on the screen.
  • string: A sequence of characters; the primary data type for text.
  • syntax error : An error in the source code that makes it impossible to compile
  • variable: A named storage location for values. All variables have a data type, which is declared when the variable is created.
  • void : A special return type indicating the method does not return a value.

Syntax

The word syntax refers to a programming language's rules.

Consider the English sentence, "The man goes to the shop." We cannot rephrase this sentence, as the "The shop goes to the man" It is not valid/allowed in English. We can say that it has "broken" an English Syntax rule.

Similarly, in the Java Programming Language, there are Syntax rules that make the language "valid". Three of them that you should be familiar with are :

  1. Java sentences are case sensitive. Consider this valid Java statement which prints out "Hello Sumir" System.out.println("Hello Sumir"); Now if we change the p from lowercase to Uppercase in printLn as per below : System.out.Println("Hello Sumir"); This will result in a Syntax Error, your entire program will not work. Even though you have spelt everything correctly, the CASE of statements is important. Not to worry though, errors are easily identified by Netbeans or JGrasp.

  2. Statements end in semicolons. You can think of this one, as English has full stops at the end of each sentence, in Java we need a semicolon. The semicolon states that we have finished with this sentence. System.out.println("Hello Sumir");

  3. Brackets need to be matched. By this it is meant, if you have an opening bracket such as ( or { you need a corresponding ) or } respectively. This will be understood more clearly later, on, but for now just take note of it.

If any of these rules are violated, your program will not compile, which is known as a Syntax Error . This process of figuring out what the error is present in the code is known as Debugging.

Increment and Decrement Operator

Operator Use Description Example
++ x++ Increases by one x++;
-- x-- Decreases by one x--;

The increment operator increases its a variables by one.

// these two statements do the same thing
x = x + 1;
x++;

The decrement operator decreases its operand by one.

// these two statements do the same thing
x = x - 1;
x--;

The increment and decrement operators can take the form of prefix and postfix. The english definitions for these two terms :

  • A prefix is a element used in the very beginning of a word
  • A postfix is a element used at the end of a word. (also known as suffix)

In these two statements, a value of 1 is added to the variable x :

++x; // prefix
x++; // postfix

Although these two statements appear to be doing the same thing, there is a subtle difference between them.

  • If you use ++ operator as prefix like ++x – then the value of x is incremented by 1 then it returns the value.
  • If you use the ++ operator as postfix like x++ - then the value of x is returned first, the x is incremented by 1.

Increment Decrement Program

public class increment_decrement {
    public static void main(String[] args) {
        int x = 100;
        int y = 100;

        int a = x++;
        int b = y--;

        System.out.println("a:" + a); // 100
        System.out.println("b:" + b); // 100

        int s = 100;
        int t = 100;

        int c = ++s;
        int d = --t;
        
        System.out.println("c:" + c); // 101
        System.out.println("d:" + d); // 99

        System.out.println("x:" + x + " | " + "y:" + y + " | " + "s:" + s + " | " + "t:" + t ); // x:101 | y:99 | s:101 | t:99
    }
}

Logical Operator

Logical operators are used to check whether an expression is true or false.

An expression is a combination of variables, operators, and values that represents a single value.

Logical Operator Program

public class logical_operator {
    public static void main(String[] args) {
        // && operator
        System.out.println((5 > 3) && (8 > 5)); // true
        System.out.println((5 > 3) && (8 < 5)); // false

        // || operator
        System.out.println((5 < 3) || (8 > 5)); // true
        System.out.println((5 > 3) || (8 < 5)); // true
        System.out.println((5 < 3) || (8 < 5)); // false

        // ! operator
        System.out.println(!(5 == 3)); // true
        System.out.println(!(5 > 3)); // false
    }
    
}

Flow Control

In computer programming, we use the if statement to control the flow of the program. For example, if a certain condition is met, then run a specific block of code. Otherwise, run another code.

For example assigning grades (A, B, C) based on percentage obtained by a student.

  • if the percentage is above 90, assign grade A
  • if the percentage is above 75, assign grade B
  • if the percentage is above 65, assign grade C

There are four types of if-statements:

  1. if statement
  2. if-else statement
  3. if-else-if statement
  4. nested-if

If-statement

The if statement consists a test Expression.

  • If the test Expression is true, the "body" code gets executed.

Switch Statements

A switch statement allows a variable to be tested for against a list of values. Each value is called a case, and the variable gets tested against each of the cases.

switch(expression) {
   case value :
      // Statements
      break;
   
   case value :
      // Statements
      break; 
   
   // You can have any number of case statements.
   default : // Optional
      // Statements
}

The expression is evaluated once and compared with the values of each case label.

  • If there is a match, the corresponding code after the matching case label is executed.
         int number = 123;
     switch (number) {
         case 1 : 
             System.out.println("1"); // 1 gets compared against 123, not a match there nothing printed
             break;
         case 123:
             System.out.println("123"); // 123 gets compared against 123, its a match "123" printed.
             break;
     }
  • If there is no match, the code after default: is executed
         int number = 123;
     switch (number) {
         case 1 : 
             System.out.println("1"); 
             break;
         case 2:
             System.out.println("2"); 
             break;
         default:
             System.out.println(number + " is not found"); // default code executed, 123 is not found in the switch
     }
  • The break statement is used to terminate the case in the switch statement
  • If the break statement is not used, all the cases after the matching case are executed.
     int expression = 2;
  switch (expression) {
    case 1:
      System.out.println("Case 1");

      // matching case
    case 2:
      System.out.println("Case 2"); // "Case 2" will get printed

    case 3:
      System.out.println("Case 3"); // "Case 3" will get printed

    default:
      System.out.println("Default case"); // "Default case" will get printed

Type Casting

The process of converting the value of one data type (int, float, double, etc.) to another data type is known as typecasting.

There are two major types : the major 2 types.

  1. Widening Type Casting
  2. Narrowing Type Casting

Widening Type Casting

In Widening Type Casting, Java automatically converts one data type to another data type.

Converting a smaller type to a larger type size byte -> short -> char -> int -> long -> float -> double

public class wide_cast_int_double {
    public static void main(String[] args) {
        int num = 10; // create an integer
        System.out.println("Integer Value: " + num);
        
        double dNum = num;
        System.out.println("Double value: " + dNum);
    }
}

In the above example, we are assigning the int type variable named num to a double type variable named data.

Here, the Java first converts the int type data into the double type. And then assign it to the double variable.

In the case of Widening Type Casting, the lower data type (having smaller size) is converted into the higher data type (having larger size). Hence there is no loss in data. This is why this type of conversion happens automatically.

This is also known as Implicit Type Casting.

Narrowing Type Casting

In Narrowing Type Casting, we manually convert one data type into another using the parenthesis.

Converting a larger type to a smaller size type double -> float -> long -> int -> char -> short -> byte

public class narrow_cast_double_int {
    public static void main(String[] args) {
        // create double type variable
        double num = 10.99;
        System.out.println("The double value: " + num); //10.99

        // convert into int type
        int data = (int) num;
        System.out.println("The integer value: " + data); // 10
    }
}

In the above example, we are assigning the double type variable named num to an int type variable named data.

Notice the line,

int data = (int)num;

Here, the int keyword inside the parenthesis indicates that that the num variable is converted into the int type.

In the case of Narrowing Type Casting, the higher data types (having larger size) are converted into lower data types (having smaller size). Hence there is the loss of data. This is why this type of conversion does not happen automatically.

Note: This is also known as Explicit Type Casting.

Methods

A method is a block of code that performs a specific task.

In Java, there are two types of methods:

  • User-defined Methods: We can create our own method based on our requirements.
  • Standard Library Methods: These are built-in methods in Java that are available to use.

Methods are also known as functions.

User Defined Method

The syntax to declare a method is:

public static returnType methodName() {
  // method body
}

For now we are not going to explain the public and static keywords, you will use them in all your user defined methods. Later on when we cover objects and classes, these topics will be explained thoroughly.

  • returnType : It specifies what type of value a method returns For example if a method has an double return type then it returns an double value.
     public static double pi()
  {
      return 3.14159265359;
  }
    If the method does not return anything, we use the void keyword, which specifies that the method does not return anything.
    public static void newLine()
  {
      System.out.println();
  }
  • methodName - It is an identifier that is used to refer to the particular method in a program. An identifier is nothing but a user defined name assigned to an element in a program. For example : String sName; sName being the identifier.
  • methodBody - It includes the programming statements that are used to perform some tasks. The method body is enclosed inside the curly braces { }.

Note to call a method, you have to make use of ().

System.out.println(); // println is a system-defined method. Methods are called by their name/identifier followed by the ()
